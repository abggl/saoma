# 扫码查重小程序 - 导入导出功能技术规范

## 一、钉钉平台技术规范调研

### 1.1 文件操作 API 限制

根据钉钉开放平台文档，小程序文件操作有以下限制：

| API | 用途 | 限制说明 |
|-----|------|---------|
| `dd.saveFile` | 保存文件到本地 | 参数为 `tempFilePath`，不是 `filePath` |
| `dd.openDocument` | 打开文件预览 | 支持格式：doc, xls, ppt, pdf, txt |
| `dd.env.USER_DATA_PATH` | 获取用户数据目录 | 真机：`ttfile://user/`，开发工具：`https://usr/` |
| `dd.env.BIZ_PATH` | 获取业务数据目录 | 推荐用于下载文件 |
| `dd.chooseAttachment` | 选择文件 | `type: 'file'` 选择文档类文件 |

### 1.2 Excel 文件处理方案

钉钉小程序原生不支持直接生成 Excel (.xlsx) 文件，可行方案：

1. **CSV 格式（推荐）**：
   - 优点：纯文本，无需第三方库，兼容性好
   - 缺点：不支持样式、公式、多 Sheet
   - 钉钉打开方式：使用 `fileType: 'txt'` 预览

2. **SheetJS 库方案**：
   - 优点：支持真正的 Excel 格式
   - 缺点：增加包体积 (~500KB)，需要处理编码
   - 限制：钉钉小程序不支持直接运行 Node 库

3. **服务端生成方案**：
   - 优点：功能完整，支持复杂格式
   - 缺点：需要后端服务，依赖网络

**本系统采用 CSV 方案**，原因：
- 数据简单，无需复杂格式
- 导入导出速度快
- 兼容性好，可用 Excel/WPS 打开

## 二、导出功能设计

### 2.1 导出数据字段

导出 CSV 包含以下字段：

| 序号 | 字段名 | 数据类型 | 说明 | 示例 |
|-----|--------|---------|------|------|
| 1 | 序号 | Number | 数据行号 | 1, 2, 3... |
| 2 | 编码 | String | 商品编码 | TEST001 |
| 3 | 备注 | String | 商品备注 | 测试商品 |
| 4 | 提交时间 | DateTime | 记录创建时间 | 2024-01-15 10:30:00 |
| 5 | 提交人 | String | 提交者姓名 | 张三 |
| 6 | 图片数量 | Number | 附件图片数量 | 2 |
| 7 | 图片路径 | String | 图片路径列表（分号分隔） | /path/1.jpg;/path/2.jpg |

### 2.2 图片数据处理

**当前方案**：
- 图片存储在小程序本地临时路径
- 导出时仅记录图片路径
- 图片无法随 CSV 一起导出（CSV 不支持二进制附件）

**优化建议（未来版本）**：
1. 图片上传至云存储（阿里云 OSS）
2. 导出时包含图片 URL
3. 导入时根据 URL 重新下载图片

### 2.3 CSV 格式规范

```csv
序号,编码,备注,提交时间,提交人,图片数量,图片路径
"1","TEST001","测试商品","2024-01-15 10:30:00","张三","2","/path/1.jpg;/path/2.jpg"
"2","TEST002","示例商品","2024-01-15 11:00:00","李四","0",""
```

**格式要求**：
- 编码：UTF-8 with BOM (`\uFEFF` 头)
- 分隔符：逗号 `,`
- 换行符：`\n`
- 字段包裹：双引号 `"`（含特殊字符时）
- 特殊字符处理：逗号替换为中文逗号，换行替换为空格

## 三、导入模板规范

### 3.1 模板字段定义

| 字段名 | 必填 | 数据类型 | 长度限制 | 格式要求 | 示例 |
|--------|------|---------|---------|---------|------|
| 编码 | 是 | String | 1-50字符 | 字母数字组合 | TEST001 |
| 备注 | 否 | String | 0-500字符 | 任意文本 | 测试商品说明 |
| 图片 | 否 | String | 0-1000字符 | 路径或URL，分号分隔 | /path/1.jpg |

### 3.2 模板文件格式

**文件名**：`import_template.csv`

**文件内容**：
```csv
编码,备注,图片
TEST001,测试商品1,
TEST002,测试商品2,
TEST003,示例商品3,
```

**存放位置**：
- 系统生成：保存至 `dd.env.USER_DATA_PATH`
- 用户模板：用户自行创建，通过 `dd.chooseAttachment` 选择

### 3.3 数据校验规则

1. **编码校验**：
   - 不能为空
   - 不能重复（与现有数据对比）
   - 只能包含字母、数字

2. **备注校验**：
   - 可选字段
   - 超长自动截断

3. **图片校验**：
   - 可选字段
   - 导入时仅记录路径，不验证文件存在性

## 四、技术实现方案

### 4.1 导出流程

```
用户点击导出
    ↓
生成 CSV 内容（内存中）
    ↓
写入临时文件（dd.env.USER_DATA_PATH）
    ↓
保存文件（dd.saveFile）
    ↓
打开预览（dd.openDocument）
    ↓
成功：用户可查看/分享
失败：复制到剪贴板
```

### 4.2 导入流程

```
用户选择文件（dd.chooseAttachment）
    ↓
读取文件内容（fs.readFile）
    ↓
解析 CSV（按行分割，处理引号）
    ↓
数据校验（编码格式、重复检查）
    ↓
分片处理（避免阻塞UI）
    ↓
批量插入数据库
    ↓
更新 codeMap
    ↓
刷新列表显示
```

### 4.3 性能优化

**大数据量处理**：
- 导出：直接生成文件，无内存压力
- 导入：分片处理，每 100-200 条一个批次
- 查重：使用 Map 结构，O(1) 复杂度

**内存管理**：
- 使用 `setTimeout`/`requestIdleCallback` 让出主线程
- 及时释放临时变量
- 避免一次性 `setData` 大量数据

## 五、安全措施

### 5.1 数据安全

1. **敏感信息**：不导出用户 ID 等敏感字段
2. **数据验证**：导入时严格校验数据格式
3. **重复防护**：编码唯一性检查

### 5.2 文件安全

1. **路径限制**：只允许操作 `USER_DATA_PATH` 目录
2. **文件类型**：仅允许 CSV/TXT 文件
3. **大小限制**：建议单文件不超过 10MB
